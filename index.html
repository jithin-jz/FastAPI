01Introduction

What is FastAPI?

FastAPI is a modern, high-performance Python web framework for building APIs. Think of it as the backbone of your backend — when a mobile app, website, or another service needs data or wants to trigger an action, it sends an HTTP request to your FastAPI server, and FastAPI handles the rest.

The name says it all: it is fast to run, and fast to build with.

ascii diagram
Client (Browser / Mobile App)
        |
        |  HTTP Request (GET /users/42)
        v
  ┌─────────────┐
  │   FastAPI   │  ← Your backend lives here
  └─────────────┘
        |
        |  Query / Write
        v
   Database / Cache / External APIs

Why Use FastAPI?

FeatureWhat It Means for You
High PerformanceBuilt on Starlette + Uvicorn. Nearly as fast as Node.js and Go.
Async SupportHandle thousands of requests concurrently with async/await.
Auto DocumentationSwagger UI at /docs is generated automatically — no extra work.
Type SafetyPython type hints validate data at runtime via Pydantic.
Less BoilerplateWrite less code to do more, compared to Flask or Django REST.
Standards-BasedBuilt on OpenAPI and JSON Schema — industry standards.

Rule of thumb:

  • Building a REST API or microservice → FastAPI
  • Quick prototype or tiny project → Flask
  • Full-stack web app with admin panel → Django

Real-World Use Cases

  • AI/ML APIs — Wrap your ML model in a FastAPI endpoint. Data scientists love this.
  • Microservices — Small, independent services that communicate via HTTP.
  • Mobile Backends — Serve data to iOS/Android apps.
  • Internal Tools — Build data pipelines with API interfaces.
  • Real-time Apps — WebSocket support for chat, notifications, live dashboards.

02Environment Setup

Prerequisites

  • Python 3.10 or later (check with python --version)
  • pip (comes with Python)
  • A terminal / command prompt

Step 1: Create a Virtual Environment

A virtual environment isolates your project's dependencies from your system Python. Always use one.

bash
# Create the environment
python -m venv venv

# Activate it
# On macOS / Linux:
source venv/bin/activate

# On Windows:
venv\Scripts\activate

# You should now see (venv) at the start of your terminal prompt

Step 2: Install FastAPI and Uvicorn

bash
pip install fastapi uvicorn[standard]
  • fastapi — The framework itself.
  • uvicorn — The ASGI server that runs your FastAPI app. Think of it as the engine.
  • [standard] — Includes extras like websockets and python-multipart.

Step 3: Create a Requirements File

Always track your dependencies:

bash
pip freeze > requirements.txt

To reinstall everything on a new machine:

bash
pip install -r requirements.txt

03First FastAPI App

Create Your First App

Create app/main.py:

python
from fastapi import FastAPI

# Create the FastAPI application instance
app = FastAPI(
    title="My First API",
    description="Learning FastAPI step by step",
    version="1.0.0"
)

# Define a route: GET request to "/"
@app.get("/")
def read_root():
    return {"message": "Hello, World!"}

# Route with a path parameter
@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}

Run the Server

bash
uvicorn app.main:app --reload
  • app.main → the Python module path (app/main.py)
  • app → the FastAPI instance inside that module
  • --reload → auto-restart when you save changes (development only)

You should see:

Understanding the Key Concepts

The FastAPI Instance

python
app = FastAPI()

This is the core of your application. All routes, middleware, and configuration are attached to this object.

Path Operations (Routes)

python
@app.get("/users")         # HTTP GET
@app.post("/users")        # HTTP POST (create)
@app.put("/users/{id}")    # HTTP PUT (full update)
@app.patch("/users/{id}")  # HTTP PATCH (partial update)
@app.delete("/users/{id}") # HTTP DELETE

Each decorator registers a function as a handler for that HTTP method + path.

04Routing & Request Handling

Path Parameters

Path parameters are parts of the URL that vary per request.

python
from fastapi import FastAPI

app = FastAPI()

# {user_id} is a path parameter — FastAPI captures and validates it
@app.get("/users/{user_id}")
def get_user(user_id: int):  # int type hint → FastAPI validates it's a number
    return {"user_id": user_id}

# Multiple path parameters
@app.get("/users/{user_id}/posts/{post_id}")
def get_user_post(user_id: int, post_id: int):
    return {"user_id": user_id, "post_id": post_id}

What happens if you pass a string instead of an int?

http
GET /users/abc
→ 422 Unprocessable Entity (FastAPI validates automatically)

Query Parameters

Query parameters come after ? in the URL. Any function parameter that is not a path parameter becomes a query parameter automatically.

python
from typing import Optional

@app.get("/items/")
def list_items(
    skip: int = 0,            # Default value = optional param
    limit: int = 10,          # Default value = optional param
    search: Optional[str] = None  # Truly optional, can be None
):
    return {
        "skip": skip,
        "limit": limit,
        "search": search
    }

Usage:

http
GET /items/                        → skip=0, limit=10, search=None
GET /items/?skip=20&limit=5        → skip=20, limit=5, search=None
GET /items/?search=laptop&limit=3  → skip=0, limit=3, search="laptop"

Request Body

For POST/PUT requests, clients send data in the request body (usually JSON). Use Pydantic models to define the expected shape.

python
from pydantic import BaseModel

class ItemCreate(BaseModel):
    name: str
    price: float
    in_stock: bool = True

@app.post("/items/")
def create_item(item: ItemCreate):
    # item.name, item.price, item.in_stock are all available
    return {"created": item}

The client sends:

json
POST /items/
{
    "name": "Laptop",
    "price": 999.99
}

FastAPI validates the body, converts it to an ItemCreate instance, and passes it to your function.

Combining Path, Query, and Body

python
@app.put("/users/{user_id}")
def update_user(
    user_id: int,            # Path parameter
    active: bool = True,     # Query parameter
    user: UserUpdate = ...   # Request body (Pydantic model)
):
    return {"user_id": user_id, "active": active, "data": user}

FastAPI figures out the source of each parameter automatically based on:

  • Path parameter → it's in the URL path {}
  • Simple type (int, str, bool) not in path → query parameter
  • Pydantic model → request body

Status Codes

python
from fastapi import status

@app.post("/users/", status_code=status.HTTP_201_CREATED)
def create_user(user: UserCreate):
    # 201 Created is returned instead of default 200 OK
    return user

@app.delete("/users/{user_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_user(user_id: int):
    # 204 No Content — response body is empty
    return None

Response Models

Use response_model to control what data is returned — this prevents accidentally leaking sensitive fields like passwords.

python
class UserCreate(BaseModel):
    name: str
    email: str
    password: str  # Sensitive!

class UserResponse(BaseModel):
    id: int
    name: str
    email: str
    # password is NOT included here

@app.post("/users/", response_model=UserResponse, status_code=201)
def create_user(user: UserCreate):
    # Even if we return an object with `password`, FastAPI will filter it out
    db_user = save_to_db(user)
    return db_user  # password field will be stripped from response

05Data Validation with Pydantic

What is Pydantic?

Pydantic is the validation library that FastAPI uses under the hood. You define data shapes using Python type hints, and Pydantic validates and converts data automatically.

Analogy: Pydantic is like a strict form with required fields, type checks, and rules. If the form isn't filled out correctly, it's rejected before processing.

Basic Pydantic Models

python
from pydantic import BaseModel, EmailStr, Field
from typing import Optional
from datetime import datetime

class UserCreate(BaseModel):
    name: str
    email: EmailStr           # Validates it's a valid email format
    age: int = Field(ge=18, le=120, description="Must be 18 or older")
    bio: Optional[str] = None

Field validators with Field():

python
from pydantic import Field

class Product(BaseModel):
    name: str = Field(min_length=2, max_length=100)
    price: float = Field(gt=0, description="Must be positive")
    quantity: int = Field(ge=0)  # ge = greater than or equal
    sku: str = Field(pattern=r"^[A-Z]{3}-\d{4}$")  # Regex pattern

Type Hints FastAPI + Pydantic Support

python
from typing import Optional, List, Dict, Union
from datetime import date, datetime
from enum import Enum

class Status(str, Enum):
    active = "active"
    inactive = "inactive"
    pending = "pending"

class Order(BaseModel):
    id: int
    status: Status                     # Enum validation
    tags: List[str] = []              # List of strings
    metadata: Dict[str, str] = {}     # Key-value pairs
    created_at: datetime               # Auto-parsed from ISO string
    notes: Optional[str] = None       # Can be None
    amount: Union[int, float]         # Accepts int or float

Nested Models

class Address(BaseModel):

Custom Validators

python
from pydantic import BaseModel, field_validator

class RegistrationForm(BaseModel):
    username: str
    password: str
    confirm_password: str

    @field_validator("username")
    @classmethod
    def username_must_be_alphanumeric(cls, value):
        if not value.isalnum():
            raise ValueError("Username must contain only letters and numbers")
        return value.lower()  # Normalize to lowercase

    @field_validator("confirm_password")
    @classmethod
    def passwords_must_match(cls, value, info):
        if "password" in info.data and value != info.data["password"]:
            raise ValueError("Passwords do not match")
        return value

Real-World Example: E-commerce Product API

python
from pydantic import BaseModel, Field, HttpUrl
from typing import List, Optional
from enum import Enum
from datetime import datetime

class Category(str, Enum):
    electronics = "electronics"
    clothing = "clothing"
    books = "books"
    food = "food"

class ProductImage(BaseModel):
    url: HttpUrl
    alt_text: str = ""

class ProductBase(BaseModel):
    name: str = Field(min_length=2, max_length=200)
    description: Optional[str] = None
    price: float = Field(gt=0)
    category: Category
    images: List[ProductImage] = []

class ProductCreate(ProductBase):
    # Used when creating a product (no id yet)
    pass

class ProductUpdate(BaseModel):
    # All fields optional for partial updates
    name: Optional[str] = Field(None, min_length=2, max_length=200)
    price: Optional[float] = Field(None, gt=0)
    description: Optional[str] = None

class ProductResponse(ProductBase):
    # Returned to clients — includes server-generated fields
    id: int
    created_at: datetime
    updated_at: datetime
    is_active: bool = True

    class Config:
        from_attributes = True  # Allows creating from SQLAlchemy ORM objects

app/services/user_service.py — Business logic:

python
from sqlalchemy.orm import Session
from app.models.user import User
from app.schemas.user import UserCreate
from app.core.security import hash_password

def get_user_by_id(db: Session, user_id: int):
    return db.query(User).filter(User.id == user_id).first()

def get_user_by_email(db: Session, email: str):
    return db.query(User).filter(User.email == email).first()

def create_user(db: Session, user_data: UserCreate) -> User:
    hashed_pw = hash_password(user_data.password)
    db_user = User(
        name=user_data.name,
        email=user_data.email,
        hashed_password=hashed_pw
    )
    db.add(db_user)
    db.commit()
    db.refresh(db_user)
    return db_user

app/routers/users.py — The HTTP layer:

python
from fastapi import APIRouter, Depends, HTTPException, status
from sqlalchemy.orm import Session
from app.schemas.user import UserCreate, UserResponse
from app.services import user_service
from app.dependencies.db import get_db

router = APIRouter(prefix="/users", tags=["Users"])

@router.post("/", response_model=UserResponse, status_code=status.HTTP_201_CREATED)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    existing = user_service.get_user_by_email(db, user.email)
    if existing:
        raise HTTPException(status_code=400, detail="Email already registered")
    return user_service.create_user(db, user)

@router.get("/{user_id}", response_model=UserResponse)
def get_user(user_id: int, db: Session = Depends(get_db)):
    user = user_service.get_user_by_id(db, user_id)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

app/main.py — Assemble everything:

python
from fastapi import FastAPI

app = FastAPI(title="My API", version="1.0.0")

app.include_router(users.router)
app.include_router(products.router)

    return {"status": "healthy"}

07Dependency Injection

What is Dependency Injection?

Dependency Injection (DI) is a pattern where a function declares what it needs, and the framework provides those things automatically.

Analogy: Think of ordering at a restaurant. You say "I'll have the pasta" — you don't go to the kitchen and cook it yourself. The restaurant (FastAPI) handles getting it to you.

Without DI:

python
@app.get("/users")
def get_users():
    db = create_database_connection()  # You manage setup every time
    auth = verify_token(request.headers["Authorization"])  # Repeated everywhere
    # ... business logic
    db.close()  # You manage teardown every time

With DI:

python
@app.get("/users")
def get_users(db: Session = Depends(get_db), current_user = Depends(get_current_user)):
    # FastAPI provides db and current_user automatically
    # ... only business logic here

Creating a Dependency

A dependency is just a function. FastAPI calls it and injects the return value.

python
from fastapi import Depends, Header, HTTPException

# Simple dependency: extract and validate an API key
def verify_api_key(x_api_key: str = Header(...)):
    if x_api_key != "my-secret-key":
        raise HTTPException(status_code=403, detail="Invalid API key")
    return x_api_key

# Use it in a route
@app.get("/protected")
def protected_route(api_key: str = Depends(verify_api_key)):
    return {"message": "Access granted"}

Database Session Dependency

The most common dependency — providing a database session per request and closing it afterward:

python
from app.database import SessionLocal

def get_db():
    db = SessionLocal()
    try:
        yield db        # Provide the session to the route handler
    finally:
        db.close()      # Always close, even if an exception occurred

# Usage in any route:
@app.get("/users/{id}")
def get_user(id: int, db: Session = Depends(get_db)):
    return db.query(User).filter(User.id == id).first()

Auth Dependency

python
from fastapi.security import OAuth2PasswordBearer
from jose import jwt, JWTError

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")

def get_current_user(token: str = Depends(oauth2_scheme), db: Session = Depends(get_db)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        user_id = payload.get("sub")
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")
    user = db.query(User).filter(User.id == user_id).first()
    if not user:
        raise HTTPException(status_code=401, detail="User not found")
    return user

# Any route that requires login:
@app.get("/profile")
def my_profile(current_user: User = Depends(get_current_user)):
    return current_user

Chaining Dependencies

Dependencies can depend on other dependencies:

python
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def get_current_user(db: Session = Depends(get_db), token: str = Depends(oauth2_scheme)):
    # get_db is called first, then its result is injected here
    ...

def get_admin_user(current_user: User = Depends(get_current_user)):
    if current_user.role != "admin":
        raise HTTPException(status_code=403, detail="Admins only")
    return current_user

@app.delete("/users/{id}")
def delete_user(id: int, admin: User = Depends(get_admin_user)):
    # Only admins can reach here
    ...

08Database Integration

SQLAlchemy Basics

SQLAlchemy is the most popular Python ORM (Object Relational Mapper). An ORM lets you interact with a database using Python objects instead of raw SQL.

Setup

bash
pip install sqlalchemy psycopg2-binary  # PostgreSQL
# or
pip install sqlalchemy                  # SQLite (no extra driver needed)

app/database.py:

python
from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker
from app.config import settings

# Database URL examples:
# PostgreSQL: "postgresql://user:password@localhost/dbname"
# SQLite:     "sqlite:///./mydb.db"
DATABASE_URL = settings.database_url

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

Defining ORM Models

python
# app/models/user.py
from sqlalchemy import Column, Integer, String, Boolean, DateTime, ForeignKey
from sqlalchemy.orm import relationship
from sqlalchemy.sql import func
from app.database import Base

from sqlalchemy.orm import Session
from app.models.user import User
from app.schemas.user import UserCreate, UserUpdate

# CREATE
def create_user(db: Session, user_data: UserCreate) -> User:
    db_user = User(**user_data.model_dump(exclude={"password"}),
                   hashed_password=hash_password(user_data.password))
    db.add(db_user)
    db.commit()
    db.refresh(db_user)  # Reload from DB to get generated fields (id, created_at)
    return db_user

# READ — single
def get_user(db: Session, user_id: int):
    return db.query(User).filter(User.id == user_id).first()

# READ — list with pagination
def get_users(db: Session, skip: int = 0, limit: int = 100):
    return db.query(User).offset(skip).limit(limit).all()

# UPDATE
def update_user(db: Session, user_id: int, user_data: UserUpdate):
    db_user = get_user(db, user_id)
    if not db_user:
        return None
    update_data = user_data.model_dump(exclude_unset=True)  # Only set fields
    for field, value in update_data.items():
        setattr(db_user, field, value)
    db.commit()
    db.refresh(db_user)
    return db_user

# DELETE
def delete_user(db: Session, user_id: int) -> bool:
    db_user = get_user(db, user_id)
    if not db_user:
        return False
    db.delete(db_user)
    db.commit()
    return True

from datetime import datetime, timedelta
from jose import jwt, JWTError
from app.config import settings

def create_access_token(data: dict, expires_delta: timedelta = None):
    payload = data.copy()
    expire = datetime.utcnow() + (expires_delta or timedelta(minutes=30))
    payload.update({"exp": expire})
    return jwt.encode(payload, settings.secret_key, algorithm=settings.algorithm)

def decode_access_token(token: str):
    try:
        return jwt.decode(token, settings.secret_key, algorithms=[settings.algorithm])
    except JWTError:
        return None

10Middleware & CORS

What is Middleware?

Middleware is code that runs on every request and every response — before your route handlers and after they return. Think of it as a checkpoint layer.

ascii diagram
Client Request
      │
      ▼
┌─────────────────┐
│   Middleware 1  │  (e.g., logging)
└────────┬────────┘
         │
┌────────▼────────┐
│   Middleware 2  │  (e.g., auth check)
└────────┬────────┘
         │
┌────────▼────────┐
│  Route Handler  │  (your function)
└────────┬────────┘
         │ (response flows back up)
┌────────▼────────┐
│   Middleware 2  │
└────────┬────────┘
         │
┌────────▼────────┐

         │
      Client

CORS (Cross-Origin Resource Sharing)

CORS is a browser security mechanism. When your React frontend (at localhost:3000) calls your FastAPI backend (at localhost:8000), the browser blocks it by default — different origins. You must tell FastAPI to allow it.

from fastapi import FastAPI

    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",  # Local frontend
        "https://myapp.com"       # Production frontend
    ],
    allow_credentials=True,
    allow_methods=["*"],          # GET, POST, PUT, DELETE, etc.
    allow_headers=["*"],          # Authorization, Content-Type, etc.
)

Custom Middleware

python
import time
import logging
from fastapi import Request

logger = logging.getLogger(__name__)

@app.middleware("http")
async def log_requests(request: Request, call_next):
    start_time = time.time()

    # Code here runs BEFORE the route handler
    logger.info(f"→ {request.method} {request.url}")

    response = await call_next(request)  # Call the actual route handler

    # Code here runs AFTER the route handler
    duration = time.time() - start_time
    logger.info(f"← {response.status_code} ({duration:.3f}s)")

    # Add a custom header to every response
    response.headers["X-Process-Time"] = str(duration)
    return response

Rate Limiting Middleware (Basic)

python
from fastapi import Request, HTTPException
from collections import defaultdict
import time

request_counts = defaultdict(list)
RATE_LIMIT = 100   # requests
WINDOW = 60        # seconds

@app.middleware("http")
async def rate_limit(request: Request, call_next):
    client_ip = request.client.host
    now = time.time()

    # Remove old requests outside the window
    request_counts[client_ip] = [
        t for t in request_counts[client_ip] if now - t < WINDOW
    ]

    if len(request_counts[client_ip]) >= RATE_LIMIT:
        raise HTTPException(status_code=429, detail="Too many requests")

    request_counts[client_ip].append(now)
    return await call_next(request)

11Background Tasks & Async

async vs sync in FastAPI

FastAPI supports both sync and async route handlers. Understanding when to use each is critical.

Use async def when:

  • Calling await-able operations (HTTP calls, async DB, etc.)
  • I/O-bound work that benefits from concurrency

Use def (sync) when:

  • CPU-bound work (data processing, ML inference)
  • Using blocking libraries like SQLAlchemy's sync driver
python
import asyncio
import httpx

# Async route — can handle many requests concurrently
@app.get("/data")
async def get_external_data():
    async with httpx.AsyncClient() as client:
        response = await client.get("https://api.example.com/data")
    return response.json()

# Sync route — FastAPI runs this in a thread pool, so it doesn't block
@app.get("/compute")
def heavy_computation():
    result = perform_heavy_calculation()  # Blocking is OK here
    return {"result": result}

The critical mistake to avoid:

python
# BAD — never call blocking code inside async def
@app.get("/bad")
async def bad_example():
    time.sleep(5)           # Blocks the entire event loop!
    requests.get("...")     # Blocking HTTP — use httpx instead!
    db.query(User).all()    # Sync DB in async context — bad!

Background Tasks

Background tasks run after the response is sent to the client. Perfect for things like sending emails, generating reports, or logging — where the user shouldn't wait.

python
from fastapi import FastAPI, BackgroundTasks
import smtplib

app = FastAPI()

def send_welcome_email(email: str, name: str):
    # This runs after response is sent — user doesn't wait
    print(f"Sending welcome email to {email}...")
    # ... actual email sending logic

@app.post("/users/", status_code=201)
def create_user(user: UserCreate, background_tasks: BackgroundTasks, db: Session = Depends(get_db)):
    db_user = user_service.create_user(db, user)

    # Schedule background task — response is sent immediately
    background_tasks.add_task(send_welcome_email, user.email, user.name)

    return db_user  # Client gets this right away, email is sent after

from fastapi import FastAPI, File, UploadFile, HTTPException
from pathlib import Path
import shutil

app = FastAPI()

UPLOAD_DIR = Path("uploads")
UPLOAD_DIR.mkdir(exist_ok=True)

@app.post("/upload/")
async def upload_file(file: UploadFile = File(...)):
    # Validate file type
    allowed_types = {"image/jpeg", "image/png", "image/gif"}
    if file.content_type not in allowed_types:
        raise HTTPException(status_code=400, detail="Only image files allowed")

    # Validate file size (5MB limit)
    content = await file.read()
    if len(content) > 5 * 1024 * 1024:
        raise HTTPException(status_code=400, detail="File too large (max 5MB)")

    # Save the file
    file_path = UPLOAD_DIR / file.filename
    with open(file_path, "wb") as f:
        f.write(content)

    return {"filename": file.filename, "size": len(content)}

# Multiple file uploads
@app.post("/upload-many/")
async def upload_multiple(files: list[UploadFile] = File(...)):
    results = []
    for file in files:
        content = await file.read()
        file_path = UPLOAD_DIR / file.filename
        with open(file_path, "wb") as f:
            f.write(content)
        results.append({"filename": file.filename})
    return results

16Best Practices

Clean Architecture Principles

1. Separation of Concerns

Each layer has one job:

ascii
Router  → Handle HTTP (parse request, return response)
Service → Business logic (rules, transformations)
Model   → Data persistence (DB operations)
Schema  → Data validation and serialization

2. Never Put Business Logic in Routers

python
# BAD — business logic in router
@router.post("/users/")
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    # All this belongs in a service!
    existing = db.query(User).filter(User.email == user.email).first()
    if existing:
        raise HTTPException(400, "Email taken")
    hashed = bcrypt.hash(user.password)
    db_user = User(name=user.name, email=user.email, hashed_password=hashed)
    db.add(db_user)
    db.commit()
    db.refresh(db_user)
    return db_user

# GOOD — router delegates to service
@router.post("/users/")
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    return user_service.create_user(db, user)

17Common Mistakes

1. Blocking Async Code

python
# BAD — time.sleep blocks the entire event loop
@app.get("/data")
async def get_data():
    time.sleep(3)  # Blocks ALL requests for 3 seconds!
    return {"data": "..."}

# GOOD — use asyncio.sleep in async context
import asyncio

@app.get("/data")
async def get_data():
    await asyncio.sleep(3)  # Only this coroutine waits
    return {"data": "..."}

2. Using requests Library in Async Routes

python
# BAD
import requests

@app.get("/external")
async def call_external():
    r = requests.get("https://api.example.com")  # Blocking!
    return r.json()

# GOOD
import httpx

@app.get("/external")
async def call_external():
    async with httpx.AsyncClient() as client:
        r = await client.get("https://api.example.com")
    return r.json()

3. Exposing Sensitive Fields in Responses

python
# BAD — password hash returned to client!
@app.get("/users/{id}")
def get_user(id: int, db: Session = Depends(get_db)):
    return db.query(User).filter(User.id == id).first()
    # SQLAlchemy object includes hashed_password

# GOOD — use response_model to filter fields
@app.get("/users/{id}", response_model=UserResponse)
def get_user(id: int, db: Session = Depends(get_db)):
    return db.query(User).filter(User.id == id).first()
    # hashed_password is NOT in UserResponse, so it's filtered out

4. Hardcoding Configuration

python
# BAD
@app.post("/login")
def login(credentials: LoginForm):
    token = jwt.encode(data, "my-secret-key", algorithm="HS256")  # Hardcoded!

# GOOD
from app.config import settings
token = jwt.encode(data, settings.secret_key, algorithm=settings.algorithm)

5. Not Handling 404 Properly

python
# BAD — returns None which FastAPI tries to serialize (causes 500)
@app.get("/users/{id}")
def get_user(id: int, db: Session = Depends(get_db)):
    return db.query(User).filter(User.id == id).first()  # Returns None silently

# GOOD — explicit 404
@app.get("/users/{id}")
def get_user(id: int, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.id == id).first()
    if not user:
        raise HTTPException(status_code=404, detail="User not found")
    return user

6. Creating a New DB Session Per Query

python
# BAD — not using dependency injection
@app.get("/users")
def get_users():
    db = SessionLocal()      # New connection every call
    users = db.query(User).all()
    db.close()               # Easy to forget, leak connections!
    return users

# GOOD — use the dependency
@app.get("/users")
def get_users(db: Session = Depends(get_db)):
    return db.query(User).all()

7. Missing Error Handling in Background Tasks

python
# BAD — exception in background task crashes silently
def send_email(to: str):
    smtp.send(to)  # If this fails, no one knows

# GOOD — wrap with try/except and log
def send_email(to: str):
    try:
        smtp.send(to)
        logger.info(f"Email sent to {to}")
    except Exception as e:
        logger.error(f"Failed to send email to {to}: {e}", exc_info=True)

WebSockets

WebSockets allow real-time, bidirectional communication between client and server. Perfect for chat, live dashboards, or notifications.

python
from fastapi import WebSocket, WebSocketDisconnect
from typing import List

class ConnectionManager:
    def __init__(self):
        self.active_connections: List[WebSocket] = []

    async def connect(self, websocket: WebSocket):
        await websocket.accept()
        self.active_connections.append(websocket)

    def disconnect(self, websocket: WebSocket):
        self.active_connections.remove(websocket)

    async def broadcast(self, message: str):
        for connection in self.active_connections:
            await connection.send_text(message)

manager = ConnectionManager()

@app.websocket("/ws/chat")
async def websocket_chat(websocket: WebSocket):
    await manager.connect(websocket)
    try:
        while True:
            data = await websocket.receive_text()
            await manager.broadcast(f"Message: {data}")
    except WebSocketDisconnect:
        manager.disconnect(websocket)
        await manager.broadcast("A user disconnected")

Redis Caching

bash
pip install redis
python
import redis
import json
from functools import wraps

redis_client = redis.from_url("redis://localhost:6379", decode_responses=True)

def cache(expire_seconds: int = 60):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            # Build cache key from function name + arguments
            cache_key = f"{func.__name__}:{str(args)}:{str(kwargs)}"

            # Check cache first
            cached = redis_client.get(cache_key)
            if cached:
                return json.loads(cached)

            # Call the actual function
            result = await func(*args, **kwargs)

            # Store in cache
            redis_client.setex(cache_key, expire_seconds, json.dumps(result))
            return result
        return wrapper
    return decorator

@app.get("/products/")
@cache(expire_seconds=300)  # Cache for 5 minutes
async def get_products():
    # This expensive query runs once, then results are cached
    return product_service.get_all_products()
